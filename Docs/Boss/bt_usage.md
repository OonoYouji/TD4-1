# ビヘイビアツリー (BT) システム使用マニュアル

ONEngine のビヘイビアツリーシステムは、Unreal Engine (UE) の設計思想に基づいた、高機能でイベント駆動な AI 構築ツールです。C++ のグラフエディタと C# のランタイムが高度に連携し、視覚的かつ直感的な AI 開発を可能にします。

---

## 1. エディタの基本操作

### エディタの開き方
*   Engine 起動後、上部タブの **「Develop」** -> **「AI Editor」** を選択します。
*   デフォルトでは `Assets/AITrees/DefaultTree.json` が読み込まれます。

### グラフ操作
*   **ノード作成:** キャンバスの何もないところで **右クリック**。検索窓でノード名を入力して選択。
*   **接続:** ノードの `Out` ピンから別のノードの `In` ピンへドラッグ。
*   **削除:** ノードやリンクを選択して `Delete` キー。
*   **移動:** ノードのタイトルバーをドラッグ。

### ショートカットキー (生産性向上機能)
| ショートカット | 機能 |
| :--- | :--- |
| `Ctrl + Z` | **Undo (元に戻す):** ノードの移動、作成、削除、プロパティ編集を戻します。 |
| `Ctrl + Y` | **Redo (やり直し):** Undo した操作をやり直します。 |
| `Ctrl + C` | **Copy:** 選択したノード（とその設定）をクリップボードにコピーします。 |
| `Ctrl + V` | **Paste:** クリップボードのノードをキャンバスに貼り付けます。 |
| `Ctrl + D` | **Duplicate:** 選択したノードをその場に複製します。 |

## 2. モジュールシステム (Decorator & Service)

ノード単体で条件を分岐させるのではなく、既存のノードに「機能」をアタッチして拡張します。

### Decorator (条件)
*   **役割:** ノードの実行可否を判定し、条件が崩れた際の中断を管理します。
*   **追加:** ノードを右クリック -> **「Add Decorator」**。
*   **Observer Aborts (監視による中断):**
    *   `None`: 中断しない。
    *   `Self`: 実行中に条件が false になったら、自分自身（と下位ツリー）を即座に強制終了します。

### Service (定期処理)
*   **役割:** ノードが実行されている間、一定間隔（Interval）で実行されるバックグラウンド処理。
*   **追加:** ノードを右クリック -> **「Add Service」**。
*   **主な用途:** ターゲットとの距離計算、Blackboard変数の更新など。

---

## 3. Blackboard (ブラックボード)

AI 全体で共有する「記憶（変数）」のストレージです。

### 変数の管理
*   左パネルの **「Add Variable」** から追加。
*   `Int`, `Float`, `Bool`, `Vec3`, `String` の型をサポート。
*   **Undo 対応:** 変数名や初期値の編集も `Ctrl+Z` で戻せます。

### ノードとの連携
*   C# ノードクラスで `[BlackboardKey]` 属性を付けた文字列フィールドは、エディタ上で自動的にドロップダウン（Blackboard 変数選択）になります。

---

## 4. デバッグ機能

### リアルタイム・ハイライト
*   ゲーム実行中、ノードの枠線が状態に応じて色分けされます：
    *   **黄色:** 実行中 (Running)
    *   **緑:** 成功 (Success)
    *   **赤:** 失敗 (Failure)

### インスペクター
*   ノードを選択すると、右パネルに C# で定義された公開フィールドが表示されます。
*   **実行時書き換え:** ゲーム実行中にインスペクターの値を変更すると、即座に C# 側の AI の挙動に反映されます。

## 5. 実践例：プレイヤーを検知して追跡する

1.  **Blackboard 設定:** 
    *   `TargetName` (String) = "Player"
    *   `IsDetected` (Bool) = false
2.  **ツリー構成:**
    *   `Sequence` ノードを作成。
        *   **SensingService** をアタッチ。
            *   `targetNameKey` = `TargetName`
            *   `resultBoolKey` = `IsDetected`
            *   `detectionRange` = 15.0
        *   **MoveToPlayerNode** を子ノードに作成。
            *   **BlackboardDecorator** をアタッチ。
                *   `keyName` = `IsDetected`
                *   `AbortPolicy` = **`Self`**

**動作結果:** 15m 以内にプレイヤーが入ると `IsDetected` が true になり追跡を開始。プレイヤーが 15m 外に逃げると Decorator が検知し、即座に追跡（MoveToPlayerNode）を中断してボスが停止します。
