# 仕様と実装における懸念点・未定義事項

現在の仕様書（概要、プレイヤー、敵、オブジェクト）を実装レベルに落とし込むにあたり、設計が不可能、または不鮮明な箇所を洗い出したリストです。
今後の仕様策定の参考としてください。

## 1. 弾の吸着と移動の物理的矛盾
* **「留まる」状態の定義と解除**: 
  * 吸着した弾がいつ、どのような条件で「留まる」のをやめるのかが不明。（時間経過？プレイヤーによる回収？特定の「終結」アクション？）
* **「横移動すると吸着し一緒に移動する」の力学的挙動**:
  * 吸着対象の壁や敵が「回転」した場合、弾の向きも同期して回転するのか？
  * 吸着したオブジェクトが破壊・消滅した場合、弾はそのまま空中に残るのか、落下するのか、消滅するのか？
* **多重吸着の優先順位**:
  * 弾が「敵」と「壁」の境界線や狭い隙間に当たった場合、どちらに吸着するかの優先順位。

## 2. Uターン挙動の幾何学的定義
* **Uターンの「回転方向」**:
  * Uターン開始時、左右どちらに旋回するのか、あるいは「プレイヤーがいる方向」等、計算に基づくのか。
  * 旋回方向に障害物がある場合、壁にめり込む（スタックする）のを防ぐ処理が必要。
* **Uターン中の当たり判定**:
  * Uターン（旋回中）の軌道上に壁や敵がいた場合、「吸着」や「ダメージ判定」は通常通り行われるのか。

## 3. 敵AIの行動優先度とスタック
* **「一定距離を保つ」と「円周移動」の競合**:
  * プレイヤーが急接近した際、円周移動（横への回避）と距離をとる（後退）のどちらを優先するのか。
  * 壁際や角に追い詰められた際の回避ロジック（壁に埋まる問題の対策）。
* **HP回復のトリガー詳細**:
  * 「ダメージを受けてから一定時間は回復しない」について、回復停止期間中に微小なダメージを受けた場合、タイマーはリセットされるのか。

## 4. プレイヤーの「腕」と「当たり判定」
* **腕の物理的な扱い**:
  * 「腕もすり抜けない」とあるが、腕が壁に当たった際、プレイヤー本体の移動も制限される（剛体として振る舞う）のか、腕が壁に沿って滑るなどの補正がかかるのか。剛体の場合、狭い通路での操作性が悪化する懸念あり。
* **発射口の壁抜け問題**:
  * 「発射場所に当たり判定はいらない」とあるが、プレイヤーが壁に密着して撃った際、銃口が壁の向こう側に突き出て、壁の向こうに弾が生成される（壁抜け撃ち）現象を防ぐ処理が必要。

## 5. 「概要」にある「終結させたくなるシステム」
* **システム的な裏付けの欠如**:
  * コンセプトにある「終結させたくなるシステム」について、現状の仕様書に具体的な「終結アクション」（ボタン操作、弾の一斉爆発、一斉回収など）の記述がない。
  * この「終結」の仕様が決まらないと、弾を吸着させて溜め込む意味やメリットのバランス調整が困難。
