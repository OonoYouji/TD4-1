# Player状態管理

このドキュメントはC#におけるプレイヤーの実装について記述したものです。プレイヤーの状態は、複数のスクリプトに分散して管理されています。

## コアロジックと体力 (`PlayerCore.cs`)

*   **役割**: プレイヤーの体力、ダメージ処理、死亡時の挙動を管理する中心的なスクリプトです。
*   **状態**:
    *   **体力 (`maxHp`, `currentHp`)**: プレイヤーの最大体力と現在の体力を保持します。
    *   **死亡状態 (`IsDead`)**: `currentHp` が0以下になると `true` となり、死亡状態を示します。
*   **機能**:
    *   `TakeDamage`: ダメージを受け、体力を減少させます。
    *   `OnDead`: 体力が0になった際に呼び出され、「GameOver」シーンへの遷移をトリガーします。
    *   衝突処理: 敵の弾 (`"EnemyBullet"`) との衝突時にダメージを受ける処理を実装しています。
    *   **特殊なエンティティ**: メインのプレイヤーエンティティとは別に存在し、プレイヤーに追従することで「当たり判定のあるコア」として機能します。

## 移動、入力、および主要な攻撃 (`Player.cs`)

*   **役割**: プレイヤーの移動、入力の処理、および主要な攻撃メカニズムを管理します。
*   **状態**:
    *   **移動速度 (`maxMoveSpeed`)**: プレイヤーの最大移動速度。
    *   **回転速度 (`rotationSpeed`)**: 目標方向へ回転する速さ。
    *   **現在の向き (`currentYaw`)**: Y軸周りの現在の回転角度。
    *   **入力モード (`isGamepadMode`)**: ゲームパッド入力か、キーボード/マウス入力かを追跡します。
    *   **発射クールダウン (`fireInterval`, `fireCooldownTimer`)**: 主要な攻撃の発射間隔とクールダウンタイマー。
*   **機能**:
    *   キーボードまたはゲームパッド入力に基づいたプレイヤーの移動処理。
    *   マウス位置またはゲームパッドのスティック入力に基づいたプレイヤーの回転処理。
    *   クールダウンを持つ主要な発射メカニズム。
    *   ゲームパッド入力とキーボード/マウス入力モード間の切り替え。

## 副次的な攻撃 (`PlayerInputHandler.cs`)

*   **役割**: 「Uターン」弾のような副次的な攻撃メカニズムを管理します。
*   **機能**:
    *   特定の入力 (`launchKey`) に紐付けられた副次的な発射メカニズム。
    *   発射間隔 (`launchInterval`) とタイマー (`launchTimer`) を持ちます。

## UI (`UiGauge.cs`)

*   **役割**: プレイヤーの状態をUIに反映させます（主にHPバー）。
*   **機能**:
    *   `PlayerCore.cs` の `CurrentHpRatio` に基づいてUIゲージ（HPバー）を更新します。
    *   ゲージの幅をプレイヤーの現在の体力に合わせてスケールします。
